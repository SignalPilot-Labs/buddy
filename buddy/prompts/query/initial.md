Begin your improvement pass on the codebase.

Step 1: Explore
- Read the project's README, CLAUDE.md, or package.json/pyproject.toml to understand the project
- Explore the directory structure to understand architecture
- Check for existing tests and their coverage

Step 2: Identify
- Find the highest-impact improvements: bugs, security issues, missing tests, code quality problems
- Prioritize: security > correctness > error handling > code quality > tests

Step 3: Implement
- Work through improvements one at a time
- For each: understand → plan → build → review → commit → push
- Keep changes small and focused — one logical change per commit

Take your time, be thorough, and commit often.
